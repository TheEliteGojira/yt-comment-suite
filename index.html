AboutThis App.
How the archiver works, what it costs, and how to get the most out of it.
What is this?

TubeArchiver is a self-contained browser app for fetching, browsing, and exporting every public comment under any YouTube video. It runs entirely in your browser — no servers, no accounts, no cloud storage. Your comments and API key never leave your machine. ||It was written by Claude, btvv, so aiGODS stay vvinning||

It is split into two tools that work together:

  • Archiver — connects to the YouTube Data API v3 with your key, paginates through all comment threads and replies, and streams them into a live preview. Estimate the quota cost before committing to a full fetch. When done, open the archive directly in the Viewer or export it to a file.
  • Viewer — browse, search, sort, filter, and export comments. Load data from the Archiver in memory, drop a saved JSON file, or fetch a second video directly from the Viewer and merge it in alongside the first.
Getting an API Key

The app uses YouTube's official Data API v3, which is free up to a daily quota limit. You supply your own key — you control your own quota.

  1. Go to console.cloud.google.com
  2. Create a new project (or select an existing one)
  3. Click Enable APIs & Services → search for YouTube Data API v3 → Enable
  4. Go to Credentials → Create Credentials → API Key
  5. Copy the key (starts with AIza) and paste it into the Archiver tab

Your key is saved in your browser's local storage so you only need to enter it once. It is never sent anywhere except directly to Google's API servers.

API Quota Costs

YouTube gives every project 10,000 units per day for free. Quota resets at midnight Pacific Time. Use the Estimate button next to the URL field to check the cost of any video before fetching.

OperationUnitsNotes
Fetch video metadata / Estimate 1 Once per fetch or estimate
One page of comment threads 1 Up to 100 top-level comments per page
One page of replies (overflow) 1 Only when a thread has more than 5 replies
Threads with ≤5 replies are bundled free in the thread response — no extra call needed.

Practical estimates:

Video sizeApprox. commentsApprox. units
Small ~500 ~6–10
Medium ~5,000 ~55–80
Large ~50,000 ~550–900
Viral ~200,000 ~2,100–4,000
Massive ~500,000+ May exceed daily quota

Reply-heavy videos cost more — each thread with more than 5 replies requires its own API call. A video with long reply chains can cost 2–3× more than the top-level count alone suggests. If you hit the limit mid-fetch, the app shows a clear error. You can export whatever was captured and continue the next day, or use a second key from a different Google Cloud project.

Archiver
  • Estimate before fetching: Click Estimate next to the URL field to fetch just the video metadata (1 unit) and see the projected quota cost before committing to a full fetch.
  • Live preview: Comments stream into the preview pane as they are fetched, capped at 100 visible items. The full archive is always available in the Viewer regardless of preview size.
  • Stop and resume: The Stop button halts after the current page finishes and disappears once the fetch is complete. You can export or open in the Viewer with whatever was captured.
  • Include replies toggle: Disable reply fetching to reduce quota usage on reply-heavy videos. Fetches top-level comments only.
  • Export formats: JSON (re-importable, nested structure), CSV (spreadsheet-compatible, flat), and plain TXT. All exports use the original unformatted text — no HTML tags.
  • Open in Viewer: Passes the archive directly to the Viewer in memory — no file download required.
Viewer
  • Search: Filters comments and replies in real time. Matches are highlighted; non-matching replies in a matched thread are dimmed so you can read the full conversation context. Search operates on plain text — never matches inside HTML formatting.
  • Sort: Newest first, oldest first, or most liked. Applies to top-level comments; replies always stay in chronological order under their parent.
  • Filter toggles: Show or hide top-level comments and/or replies independently. Combine with search for fine-grained results.
  • Date range: Narrow the feed to comments posted between two dates. Clears with the ✕ button.
  • Pin / bookmark: Click ☆ on any top-level comment to pin it. Use the ★ Pinned filter to view only pinned comments. Pins are session-only and clear on reset.
  • Multi-archive merge: Click ⊕ Merge archive to bring in a second archive alongside the first. Choose Fetch from YouTube to enter a URL and pull comments directly (uses your saved API key, includes an Estimate option), or Load from disk to import a saved .json file. Merged threads are deduplicated by comment ID. Each merged source appears as a chip in the source bar — click × on any chip to remove that source and its threads from the current session. The title bar shows a +N archives count while sources are active.
  • Word frequency: The collapsible Word Frequency panel (between the meta bar and controls) shows the top 30 words across all loaded comments, ranked by count with a proportional bar. Computed once on first open and cached until a new archive is loaded.
  • Export filtered results: Always visible once an archive is loaded. Label reads "All (N):" or "Filtered (N):" depending on active filters. Exports the current view as JSON, CSV, or TXT. Merged archives export as a single unified file.
  • Permalinks ↗: Every comment and reply card links directly to that comment on YouTube in a new tab.
  • Copy ⧉: Copies the plain text of any comment or reply to the clipboard. Confirms with ✓ for 1.5 seconds.
  • Timezone: All timestamps are stored as UTC. Switch timezone freely — the Viewer converts on the fly without reloading.
  • Author profiles: Click any commenter's name to open a profile panel showing all their activity in the loaded archive — comment count, reply count, total likes, and full history. Hover a name for a quick tooltip. Click the channel name in the meta bar to see the video author's comments.
  • Video meta bar: Shows the video thumbnail, linked title, view and like counts, and a collapsible description. All metadata is stored in the JSON and survives re-import.
  • Profile picture expiry: Avatar URLs are captured at fetch time and may expire if a user changes their picture. Failed avatars show a grey placeholder — re-fetching the video captures the current picture.
Running & Deploying

The app must be served over HTTP — opening index.html as a local file causes CORS errors. The quickest local server: 

cd yt-comment-suite
python3 -m http.server 8080
# Then open: http://localhost:8080

To share publicly, drag the yt-comment-suite/ folder onto app.netlify.com/drop for a free URL in about 30 seconds. GitHub Pages also works and redeploys automatically on every git push.
