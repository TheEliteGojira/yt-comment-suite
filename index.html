AboutThis App.
How the archiver works, what it costs, and how to get set up.
What is this?

Tube Archiver is a self-contained web app that lets you fetch, browse, and export every public comment under any 'tube video. It runs entirely in your browser — no servers, no accounts, no cloud storage. Your comments and API key never leave your machine. It was written by Claude, btvv, so aiGODS stay vvinning

It is split into two tools that work together:

  • Archiver — connects to the YouTube API with your key, paginates through all comment threads and replies, and streams them into a live preview. When done, you can open the archive directly in the Viewer or export it to a file.
  • Viewer — lets you browse, search, sort, and filter the fetched comments. You can load data directly from the Archiver (no download needed) or drop a previously exported JSON file.
Getting an API Key

The app uses YouTube's official Data API v3, which is free up to a daily quota limit. You supply your own key, so you control your own quota.

  1. Go to console.cloud.google.com
  2. Create a new project (or select an existing one)
  3. Click Enable APIs & Services → search for YouTube Data API v3 → Enable
  4. Go to Credentials → Create Credentials → API Key
  5. Copy the key (it starts with AIza) and paste it into the Archiver tab

Your key is saved in your browser's local storage so you only need to enter it once. It is never sent anywhere except directly to Google's API servers.

API Quota Costs

YouTube gives every project 10,000 units per day for free. Quota resets at midnight Pacific Time. Here is exactly what each operation costs:

Operation Units Notes
Fetch video title/metadata 1 Done once per fetch, at the start
One page of comment threads 1 Up to 100 top-level comments per page
One page of replies (overflow) 1 Only triggered when a thread has more than 5 replies
Threads with ≤5 replies are included free inside the thread response — no extra call needed.

Practical estimates:

Video size Approx. comments Approx. units used
Small ~500 ~6–10
Medium ~5,000 ~55–80
Large ~50,000 ~550–900
Viral ~200,000 ~2,100–4,000
Massive ~500,000+May exceed daily quota

Reply-heavy videos cost more because each thread with more than 5 replies requires its own API call. A video where most top-level comments have long reply chains can cost 2–3× more than the thread count alone suggests.

If you hit the quota limit mid-fetch, the app will show a clear error message. You can export whatever was captured before the limit was hit, then continue the next day with a fresh quota — or use a second API key from a different Google Cloud project.

Workflow Tips
  • No download needed: After fetching, click "→ Open in Viewer" to browse immediately. The data passes in memory — no file created.
  • Save for later: Export to JSON to archive a video's comments permanently. Drop the file back into the Viewer at any time without re-fetching.
  • Searching replies: When you search, threads with a matching reply are shown with all their replies visible — matching ones are highlighted, non-matching ones are dimmed so you see the full conversation context.
  • Sort by Most Liked: Useful for finding the community's top reactions quickly. Sorting applies to top-level comments; replies always stay in chronological order under their parent.
  • Timezone: All timestamps are stored as UTC in the JSON. The Viewer converts them to whichever timezone you select — you can switch freely without re-loading.
  • Stopping early: The Stop button halts after the current page finishes. You can still export or open in the Viewer with whatever was fetched up to that point.
  • Author profiles: In the Viewer, click any commenter's name to open a profile panel showing all of their activity in the loaded archive — comment count, reply count, total likes, and a full history. You can also click the channel name in the video info bar to see the video author's comments. Note that YouTube display names are not unique; the panel is scoped to the current archive only and makes no identity claims across videos.
  • Profile picture expiry: Avatar URLs are fetched live from Google's CDN at archive time and stored in the JSON. They may expire or become outdated if a user later changes their profile picture. If an avatar fails to load, a grey placeholder is shown instead. Re-fetching the video will capture the current picture.
Running & Deploying

Because the app calls the YouTube API, it must be served over HTTP — opening index.html as a local file will cause CORS errors. The quickest local server: 

cd yt-comment-suite
python3 -m http.server 8080
# Then open: http://localhost:8080

To share with others, drag the yt-comment-suite/ folder onto app.netlify.com/drop for a free public URL in about 30 seconds. No account required. GitHub Pages is also free and redeploys automatically on every git push.
